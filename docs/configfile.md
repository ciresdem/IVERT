# IVERT Configuration

IVERT's configuration is split into two files:

- **`ivert_defaults.ini`** — shipped with IVERT, contains default values for all settings. Do not edit this file; it will be overwritten when IVERT is upgraded.
- **User config file** — a personal override file (default location: `~/.ivert/user_config.ini`) where you set any values you want to change from the defaults. This file is created automatically the first time you change a setting.

Any setting in the user config file takes precedence over the corresponding default.

---

## Viewing current settings

```
ivert setup list
```

This lists every configurable setting, its current value, and whether the value comes from the defaults (`[default]`) or your user config (`[user]`).

---

## Changing a setting

```
ivert setup option_name=new_value
```

For example, to move your data directory to an external drive:

```
ivert setup user_data_directory=/mnt/external/ivert
```

You can set multiple values at once:

```
ivert setup cache_directory=/fast_ssd/ivert/cache icesat2_granules_directory=/fast_ssd/ivert/granules
```

If the value contains spaces or special shell characters, wrap it in quotes:

```
ivert setup user_data_directory="/home/my user/ivert data"
```

The first time you change any setting, IVERT creates your user config file at the location specified by `user_configfile` in `ivert_defaults.ini` (default: `~/.ivert/user_config.ini`). Subsequent changes update that same file.

---

## Using different config files for different projects

You may want separate configurations for different projects — for example, one with a local cache directory for a coastal survey and another pointing to a network drive for an archive workflow.

Create a config file for each project containing only the settings that differ from the defaults:

**`~/.ivert/coastal_survey.ini`**
```ini
[DEFAULT]
user_data_directory = ~/projects/coastal_survey/ivert
cache_directory = ~/projects/coastal_survey/ivert/cache
```

**`~/.ivert/archive_workflow.ini`**
```ini
[DEFAULT]
user_data_directory = /mnt/archive/ivert
cache_directory = /mnt/archive/ivert/cache
icesat2_granules_directory = /mnt/archive/ivert/icesat2/granules
```

### Switching with the `--config` flag

Pass `--config` to any `ivert` command to use a specific config file for that invocation:

```
ivert --config ~/.ivert/coastal_survey.ini download -73.5/-72.5/40.5/41.5
ivert --config ~/.ivert/archive_workflow.ini validate mydem.tif
```

`--config` must come immediately after `ivert`, before the subcommand.

### Switching with the `IVERT_USER_CONFIG` environment variable

Set `IVERT_USER_CONFIG` to make all `ivert` commands in a shell session use a particular config file without repeating `--config` each time:

```bash
export IVERT_USER_CONFIG=~/.ivert/coastal_survey.ini
ivert download -73.5/-72.5/40.5/41.5
ivert validate mydem.tif
```

To revert to the default config file, unset the variable:

```bash
unset IVERT_USER_CONFIG
```

### Priority order

When IVERT starts, the user config path is resolved in this order:

1. `--config PATH` CLI flag
2. `IVERT_USER_CONFIG` environment variable
3. `user_configfile` setting in `ivert_defaults.ini` (default: `~/.ivert/user_config.ini`)

---

## Reverting to default settings

To restore all settings to their defaults, delete your user config file:

```bash
rm ~/.ivert/user_config.ini
```

IVERT will fall back entirely to `ivert_defaults.ini` on the next run.
